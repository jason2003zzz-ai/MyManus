# GitHub 仓库与版本发布

用于检查 Git 状态、整理变更、提交代码、推送分支、处理 Pull Request、Issue 和 CI 结果。所有操作必须明确仓库、分支、变更范围和远端目标。

## 执行流程

1. 先查看仓库根目录、当前分支、远端地址和工作区状态，区分用户修改与本轮修改。
2. 提交前检查 diff，排除密钥、缓存、大型临时文件和无关产物。
3. 运行与变更相关的测试，提交信息应准确描述行为变化。
4. 推送、创建 PR、合并、关闭 Issue 或发布 Release 前确认用户已授权对应外部操作。
5. 完成后返回分支、提交哈希、PR 或 Release 链接以及测试结果。

## 安全要求

- 禁止使用 `git reset --hard` 等破坏性命令处理未知修改。
- 不得强推受保护分支，除非用户明确要求并确认风险。
- 不把本地配置、Token、Cookie 或 `.env` 提交到仓库。
