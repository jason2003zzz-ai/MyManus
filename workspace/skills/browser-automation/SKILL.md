# 浏览器自动化与网页交互

用于需要打开网站、读取页面、点击按钮、填写表单、上传文件、翻页、登录态操作或截图验证的任务。优先使用 Playwright MCP，并以页面可见状态和操作后的证据作为完成依据。

## 执行流程

1. 先读取当前页面 URL、标题和可交互元素，确认页面状态后再操作。
2. 优先依据角色、标签、可见文本和稳定属性定位元素，避免依赖易变化的坐标。
3. 每次关键点击或提交后重新读取页面，验证 URL、提示信息、按钮状态或目标内容已经变化。
4. 涉及发布、发送、删除、付款或提交时，严格使用用户指定内容；不可逆操作前确认目标和范围。
5. 任务完成前保留截图、页面文本或状态变化作为证据，并说明未能验证的部分。

## 失败恢复

- 元素不存在时先检查页面是否加载完成、是否位于 iframe、弹窗或折叠菜单中。
- 页面结构变化时重新读取 DOM，不连续重复同一失败动作。
- 登录或验证码阻塞时明确说明需要用户完成的步骤，不绕过安全验证。
