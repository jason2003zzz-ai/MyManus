# 长程任务规划与最终交付

用于“先调研资料，再生成报告，最后发送邮件并汇总交付物”一类包含明确先后关系的长程任务，也适用于跨网页、文档、表格和通信工具的复合任务。单独处理一封邮件或一个文件时使用对应专项技能；本技能负责拆分阶段、衔接结果并维护子目标、证据、产物和完成状态。

## 执行流程

1. 将任务拆分为可验证的子目标，标明依赖关系、所需工具和完成条件。
2. 每完成一步记录关键 Observation、文件路径、页面状态和未解决问题。
3. 下游步骤直接复用已确认结果，避免重复检索、重复下载和无意义操作。
4. 局部失败时先重试可恢复步骤，再调整计划；保留已完成产物并明确影响范围。
5. 结束前检查所有用户要求、文件、外部状态和证据，未完成项必须显式说明。

## 最终答案

- 先说明任务是否完成，再列出核心结果和交付物。
- 提供可访问的文件路径、链接或验证证据。
- 不把工具日志直接作为最终答案，不隐瞒失败或不确定性。
