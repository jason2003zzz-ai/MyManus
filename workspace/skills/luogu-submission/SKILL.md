# 洛谷算法题求解与提交

用于读取洛谷题目、分析算法、编写代码、本地测试、提交并检查评测结果。必须独立理解题意和约束，不能只根据样例或题号硬编码答案。

## 执行流程

1. 打开题目并完整读取题意、输入输出格式、数据范围、样例和特殊限制。
2. 推导算法与复杂度，确认时间和空间复杂度能够覆盖最大数据。
3. 编写清晰代码，处理边界、溢出、下标、空集合和多组数据等情况。
4. 在提交前进行样例、边界和自构造测试；失败时根据错误类型定位原因。
5. 选择正确语言提交，等待最终评测状态，并记录 AC、WA、TLE、RE 等结果。

## 失败恢复

- WA 时优先检查题意、边界和数据类型，不盲目重复提交。
- TLE/MLE 时重新评估复杂度和数据结构。
- 验证码、登录或平台限制应交由用户处理，不绕过安全机制。
