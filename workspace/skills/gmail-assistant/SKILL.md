# Gmail 邮箱处理

用于查找导师、同事或指定发件人的邮件，处理昨天或某个时间范围内的未读消息，并完成总结、起草回复、发送、附件处理和待办跟踪。使用 Gmail MCP 获取邮件线程和元数据，保持收件人、主题、引用关系和附件准确。

## 执行流程

1. 将用户要求转换为 Gmail 查询条件，包括发件人、收件人、主题、时间范围、是否未读和附件类型。
2. 阅读完整线程而非孤立邮件，提取背景、决定、截止时间、待回复问题和附件信息。
3. 起草回复时保持原线程语境，明确回应每个问题，不虚构承诺、时间或附件。
4. 发送、转发、归档、删除或修改标签属于外部状态变更，执行前核对目标邮件和用户授权。
5. 完成后报告处理的线程、采取的动作和仍需用户决定的事项。

## 质量要求

- 不在日志或结果中泄露 API Key、验证码和无关个人信息。
- 用户只要求“起草”时不得擅自发送。
- 附件必须核对文件名、格式和对应邮件。
