# 代码诊断、修复与测试

用于定位程序错误、分析日志、修复缺陷、补充测试和验证回归，也适用于 Python、JavaScript、接口和 Agent 工程问题。先确定根因，再进行最小范围修改。

## 执行流程

1. 阅读仓库说明、相关代码、配置和测试，复现问题并记录实际错误。
2. 沿调用链定位首个错误状态，区分根因、连带症状和环境问题。
3. 检查工作区已有修改，保留用户内容，不重置或覆盖无关文件。
4. 使用最小补丁修复根因，并为关键边界条件补充测试。
5. 运行针对性测试、静态检查和必要的集成验证，报告未运行项目及原因。

## 约束

- 不通过吞掉异常、删除测试或硬编码单个样例来制造通过。
- 不输出密钥、令牌和配置文件中的敏感值。
- 用户只要求诊断时，仅报告原因，不擅自修改代码。
